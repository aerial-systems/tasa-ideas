CI failure? Paste the log.

EnvRepro ingests raw CI output — GitHub Actions, Jenkins, CircleCI, whatever — reconstructs the exact environment the pipeline ran in, and drops you into a live debugging session with the same dependency versions and the same failure.

  • Matches exact package versions from lockfiles
  • Replays migration and seed steps from CI config
  • Captures env vars, secrets (redacted), and OS-level deps
  • Points you to the failing line with a live debugger
# Paste your CI failure — EnvRepro handles the rest
import envrepro

env = envrepro.create(
    "github.com/org/repo",
    ci_log="https://ci.example.com/build/4291",
    test="tests/test_checkout.py::test_empty_cart"
)

# 47s later…
env.open_debugger()       # Live VS Code session
env.share("alex@team.io")    # Pair-debug with a teammate
env.snapshot()               # Save state for later

FAQ

Got questions? We've got answers.

What languages and frameworks do you support?

Python, Node.js, Ruby, Go, Rust, Java, Elixir, and PHP — with first-class support for their most popular frameworks and package managers. More languages ship monthly.

How do you handle secrets and environment variables?

Secrets are encrypted at rest and injected at runtime. You can also connect your own vault (HashiCorp, AWS Secrets Manager, etc.) for Enterprise plans.

Can I use my own cloud / on-prem infra?

Yes — Enterprise plans support VPC peering for AWS, GCP, and Azure, plus an on-premises deployment option for air-gapped environments.

What happens to my environments when I'm done?

Environments auto-suspend after your session timeout and are deleted after 7 days of inactivity. Pro users can snapshot environments to preserve exact state indefinitely.

Do you support monorepos?

Absolutely. EnvRepro detects monorepo structures (Nx, Turborepo, Lerna, Bazel, Pants) and lets you target specific packages or services inside them.

How does CI log ingestion actually work?

Paste a raw CI log or provide a URL. EnvRepro parses the build steps, extracts dependency install commands, environment variables, and the failing test/command — then rebuilds that exact state in a live sandbox.
