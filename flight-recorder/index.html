How it works

Two lines in. Complete visibility out.

Wrap your agent loop once. Flight Recorder instruments every call automatically. No manual instrumentation, no schema to define.

Step 1

Install the SDK

pip install flight-recorder or npm install @flightrecorder/sdk. One dependency. No native modules.

Step 2

Wrap your agent loop

Add two lines: import the recorder, call recorder.wrap(agent). That's it. The SDK auto-detects LangChain, OpenAI, Anthropic, and custom HTTP calls.

Step 3

Open the dashboard

Every trace appears in real time. Filter by session, error, cost threshold, or eval failure. Share a trace link with your team — no login required for viewers.

Step 4

Replay failures

Click any user-reported session and watch it play back. Tool calls, prompts, context, UI state — frame by frame. Debug what actually happened, not what the logs imply.

FAQ

Questions you probably have.

Does this add latency to my agent?
The SDK buffers traces in memory and flushes them asynchronously on a background thread. In benchmarks, the median overhead is under 2ms per step. If the recorder process is unavailable, traces are written to a local disk buffer — your agent keeps running.
What frameworks do you support?
The SDK auto-instruments LangChain, LangGraph, OpenAI, Anthropic, Cohere, and any HTTP call made through fetch or requests. If you use a custom agent loop, the low-level recorder.span() API gives you manual control. Python and TypeScript are first-class. Go and Rust SDKs are in development.
Where does the data live?
By default, traces are stored in our hosted infrastructure (US East, EU West). Enterprise customers can deploy the recorder backend in their own VPC or on-prem. Data is encrypted at rest (AES-256) and in transit (TLS 1.3). We are SOC 2 Type II certified.
Can I export my traces?
Yes. Every trace can be exported as JSON, CSV, or via the REST API. Pro and above include automated daily exports to S3 or GCS. You own your data — there is no vendor lock-in.
What counts as a "trace"?
A trace is one complete agent session — from the first user message to the final response. A single trace can contain dozens of LLM calls, tool invocations, and eval checks. You only pay for the top-level session, not the individual steps inside it.
How does the failure replay work?
Flight Recorder captures the complete serialized state at each step: the prompt template, the rendered messages, the model response, the tool inputs and outputs, and the UI state snapshot. The replay engine reassembles these into a frame-by-frame playback that shows exactly what the user saw and what the agent did — no re-execution required.
